# 马里奥英语游戏 · 开发规划

## 项目概述

**项目名称**: 马里奥英语游戏 (Super Mario Word Game)
**项目类型**: HTML5 教育游戏（网页游戏）
**核心技术**: HTML5 Canvas + CSS3 + JavaScript
**游戏目标**: 通过马里奥风格的跑酷游戏学习英语词汇和句型

## 二、核心系统开发

### 2.1 游戏引擎系统

#### 2.1.1 游戏循环 (Game Loop)
```
60 FPS 主循环：
  1. 输入处理 (Input Processing)
  2. 物理更新 (Physics Update)
  3. 实体更新 (Entity Update)
  4. 碰撞检测 (Collision Detection)
  5. 相机更新 (Camera Update)
  6. 渲染 (Render)
```

#### 2.1.2 物理系统
- 重力加速度: 0.5 px/frame²
- 最大下落速度: 12 px/frame
- 水平移动速度: 4 px/frame
- 跳跃初速度: -12 px/frame
- 摩擦系数: 0.85

#### 2.1.3 碰撞检测
- AABB矩形碰撞检测
- 角色与地面的碰撞
- 角色与敌人的碰撞（踩敌人/被敌人碰到）
- 角色与道具的碰撞

### 2.2 角色系统

#### 2.2.1 马里奥 (Player)
**状态机**:
- `idle` - 站立
- `running` - 奔跑
- `jumping` - 跳跃
- `falling` - 下落
- `big` - 变大状态
- `star` - 无敌状态
- `dead` - 死亡

**属性**:
- 位置 (x, y)
- 速度 (vx, vy)
- 尺寸 (normal: 32x32, big: 32x48)
- 状态 (大小状态、星星状态)

**动画帧**:
- 站立: 1帧
- 奔跑: 4帧循环
- 跳跃: 1帧
- 变大: 1帧
- 星星: 5帧彩虹色

#### 2.2.2 敌人系统

**板栗怪 (Goomba)**
- 尺寸: 24x24
- 行为: 直线行走，遇墙反向
- 死亡: 被踩后变扁消失

**乌龟 (Koopa)**
- 尺寸: 24x32
- 行为: 直线行走
- 特殊: 被踩后变成龟壳，可踢走

**Boss库巴 (Bowser)**
- 尺寸: 200x260 (显示用)
- 行为: 固定位置，答题时出现
- HP: 3点（对应3题Boss战）

### 2.3 道具系统

| 道具 | 效果 | 动画 |
|------|------|------|
| 金币 🪙 | +10分 | 旋转、收集飞向UI |
| 蘑菇 🍄 | 变大 | 弹出、弹跳、收集 |
| 星星 ⭐ | 无敌5秒 | 发光闪烁、彩虹色 |

### 2.4 关卡系统

#### 2.4.1 关卡数据结构
```javascript
{
  id: "L1",
  world: 1,
  letter: "Aa",
  words: ["apple", "cat", "bat", "bag"],
  sentences: {
    apple: "I want to eat an ___.",
    cat: "The ___ is sleeping on the sofa.",
    // ...
  },
  platforms: [
    { type: "ground", y: 256, width: 480 },
    { type: "brick", x: 100, y: 160, width: 32, height: 32 },
    // ...
  ],
  enemies: [
    { type: "goomba", x: 200, y: 224 },
    // ...
  ],
  items: [
    { type: "coin", x: 120, y: 140 },
    { type: "mushroom", x: 150, y: 140 },
    // ...
  ],
  boss: {
    questions: 3,  // 或5（终关）
    sentences: [...]
  }
}
```

#### 2.4.2 关卡流程控制
```
开场 → [游戏段] → 单词卡1 → [游戏段] → 单词卡2 → [游戏段] → 单词卡3 → [游戏段] → 单词卡4 → [Boss战?] → 通关
```

## 三、UI系统开发

### 3.1 游戏HUD
- 顶部: 金币数 | 关卡信息 | 进度条 | 暂停按钮
- 底部: 虚拟按键（移动端）

### 3.2 单词卡片
**普通单词卡**:
- 图标 + 英文单词
- 例句填空
- 4个中文选项（1正确3错误）
- 答对动画：弹出金币
- 答错动画：选项抖动

**Boss战单词卡**:
- 红色边框
- "🔥 库巴挑战"标签
- 3个进度点
- 英文句子 + 中文翻译
- 4个英文选项

### 3.3 菜单系统
- 主菜单：开始游戏、世界选择、单词库、设置
- 世界选择：5个世界卡片（锁定/解锁状态）
- 关卡选择：每个世界的关卡列表
- 暂停菜单：继续、重来、退出

### 3.4 通关画面
- 金币雨动画
- 统计：答对数、金币数、已学字母
- 下一关按钮

## 四、5个世界设计

### 4.1 世界1：草地王国 🌿 (L1-L5)
**背景**: 蓝天白云、绿色草地
**句型**: It is a/an X. / Is it a X? / It is not a X.
**新机制**: 跳跃、顶砖块、蘑菇变大、踩敌人、星星无敌
**敌人**: 板栗怪
**配色**:
- 天空: #5c94fc
- 草地: #8BCE63
- 地面: #c84c09

### 4.2 世界2：洞穴王国 🕳️ (L6-L10)
**背景**: 深紫黑色洞穴、火焰把
**句型**: This is a X and that is a Y. / These are… / Those are…
**新机制**: 移动平台、乌龟敌人（踩变龟壳可踢）
**敌人**: 蝙蝠、蜘蛛、乌龟
**配色**:
- 天空: #1a1a2e ~ #2d2d4e
- 地面: #3d2b1e

### 4.3 世界3：海底王国 🐠 (L11-L16)
**背景**: 蓝色渐变、气泡、海草
**句型**: Look at… / Are these…? / I have a… / I don't have a…
**新机制**: 水下重力（跳高落慢）、水母（上下浮动）、河豚（膨胀敌人）
**敌人**: 水母、河豚
**配色**:
- 天空: #0277BD ~ #01579B
- 海底: #F9A825

### 4.4 世界4：天空之城 ☁️ (L17-L21)
**背景**: 浅蓝色天空、大白云
**句型**: Can X do Y? / This is a X. / She is a X. / Do you have…?
**新机制**: 浮动云平台（会消失）、风力影响跳跃
**敌人**: 云怪
**配色**:
- 天空: #E3F2FD ~ #BBDEFB ~ #90CAF9

### 4.5 世界5：库巴城堡 🏰 (L22-L26)
**背景**: 暗红色城堡、岩浆
**句型**: 综合复习
**新机制**: 全机制混合、敌人密集、火焰棒
**敌人**: 骷髅龟、火焰棒
**配色**:
- 天空: #1a0a0a ~ #2d0a0a
- 地面: #2d1b0e

## 九、技术难点

### 1. CSS像素艺术动画
**问题**: 使用CSS实现的像素角色动画需要精确控制
**方案**: 使用CSS Grid布局像素格子，通过class切换颜色

### 2. 单词卡与游戏的无缝切换
**问题**: 答题时游戏暂停，答完后恢复
**方案**: 
- 答题时暂停游戏循环
- 使用状态机控制游戏状态
- 答题完成后恢复游戏

### 3. 关卡自动生成
**问题**: 26个关卡需要可配置
**方案**: 使用JSON数据定义关卡结构，程序读取生成

### 4. Boss战交互
**问题**: 库巴需要根据答题结果移动
**方案**:
- 库巴位置与Boss HP绑定
- 答对后退，答错前进
- 3题全对触发飞走动画

## 十、开发建议

### 代码规范
- 使用ES6模块化 (import/export)
- 类名采用PascalCase
- 方法名采用camelCase
- 常量使用UPPER_SNAKE_CASE
- 注释使用中文

### 性能考虑
- 使用requestAnimationFrame
- 对象池复用（敌人、子弹）
- 视口外实体不渲染
- 图片使用CSS Sprite合并

### 调试工具
- 浏览器DevTools
- Canvas绘制边界框
- 状态打印日志
